# 플랫폼별 유의사항

| 플랫폼 | 주의점 |
|--------|--------|
| **네코챗** | JSX 렌더링 지원. 보조프롬프트(BP) 분리 가능. fetch 차단 → 모든 정보를 프롬프트 내에 포함 |
| **루나톡** | 무한프롬(800자) + 로어북(500자) 백업 권장 |
| **바베** | 출력량 지침 미적용. SVG/HTML/JSX 상태창 가능 |
| **케이브덕** | 토큰 계산 정확. 연산 기호 압축 효과 큼 |
| **티팟** | HTML 프롬 사용 가능, 사칭 방지 프롬 필요 |
| **이브/엘린** | 출력 조정용 별도 프롬 필요, free형 출력 조정 |

## 모델별 유의사항

| 모델 | 주의점 |
|------|--------|
| **저급 모델 (Gemini Flash 등)** | 연산 기호(§¶∧∨¬ 등) 인식 불가 확률 높음. 자연어로 풀어 쓸 것. 용어 치환 최소화, 예시 더 필요 |
| **고급 모델 (Gemini Pro, GPT 등)** | 연산 기호, 한자 약어 등 모두 잘 인식. 더 공격적 압축 가능 |

## 일반 원칙

- 모델이 클수록 압축에 강함 → 더 공격적 압축 가능
- 컨텍스트 윈도우가 작을수록 → 압축의 가치가 높아짐
